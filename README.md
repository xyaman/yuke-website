# yuke-website

The website for [yuke](https://github.com/xyaman/yuke). Hand-written HTML
landing at `/`, Starlight docs at `/<topic>/`.

## Develop

```sh
npm install
npm run dev          # http://localhost:4321/yuke-website/
```

## Build

```sh
npm run build        # output in ./dist
npm run preview      # serve the build locally
```

## Deploy

Pushes to `main` deploy via GitHub Pages. The site is served at
`https://xyaman.github.io/yuke-website/` until a custom domain is added.

To move to a custom domain: update `site` and remove `base` in
`astro.config.mjs`, then add a `CNAME` file under `public/`.

## Layout

```
src/
  pages/index.astro         # hand-written HTML landing
  content/docs/             # all docs as .md
    getting-started.md
    local-mode.md
    daemon-mode.md
    lua-config.md
    tools.md
    cli.md
    primitives.md
    wire-protocol.md
  assets/logo.png           # Starlight header logo (imported by astro.config.mjs)
public/
  logo.png                  # same file, served as-is for the hand-written landing
  landing.css               # stylesheet for the landing page only
  favicon.ico, favicon-32.png, favicon-16.png
  apple-touch-icon.png
  asyncapi.json             # copy of yuke/docs/asyncapi.json
```

The logo is duplicated in `src/assets/` and `public/` on purpose: Starlight
imports the former through Astro's image pipeline, the hand-written landing
references the latter as a plain `<img src>`. Both are the same 31 KB PNG.

## Logo

`src/assets/logo.png` is the source-of-truth logo (a 512×512 PNG, 64-color
palette, 31 KB). The black background is part of the artwork, so we use the
same file on both light and dark themes.

Regenerate variants from the source PNG (kept at
`~/Downloads/yuke.png`) with `magick` if needed:

```sh
magick source.png -resize 512x512 -colors 64 -dither None PNG8:logo.png
magick source.png -resize 180x180 -colors 64 -dither None PNG8:apple-touch-icon.png
magick logo.png favicon-32.png favicon-16.png favicon.ico
```

## Syncing `asyncapi.json`

`public/asyncapi.json` is a copy of `yuke/docs/asyncapi.json` (generated from
`yuke-core/` wire types). When wire types change:

```sh
# in the yuke repo
bin/generate-schema        # writes ./docs/asyncapi.json
cp docs/asyncapi.json ~/Work/yuke-website/public/asyncapi.json
```

The pre-commit hook in `yuke/` regenerates the source. Drift between the two
repos is rare; re-copy when the wire protocol changes.

## Conventions

- Hand-written landing lives in `src/pages/index.astro` and is intentionally
  outside Starlight's chrome (no `<StarlightLayout>`, no Starlight imports).
- Docs are `.md` (not `.mdx`) unless a page needs JSX components.
- The sidebar is configured manually in `astro.config.mjs`, not autogenerated,
  so doc ordering stays deliberate.
